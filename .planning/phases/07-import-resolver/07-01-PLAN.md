  <action>
    Replace the stub in `packages/core/imports.ts` with the full implementation. Follow the research patterns exactly:

    **Imports:** `node:fs` (readFileSync), `node:path` (resolve, normalize, dirname), `parse` from `./parser.js`, `LogicSpec` type from `./types.js`.

    **Public API:**
    ```
    resolveImports(spec: LogicSpec, basedir: string): ImportResult
    ```

    **Internal function:**
    ```
    resolveImportsRecursive(spec, basedir, visited: Set<string>, chain: string[]): ImportResult
    ```

    **Implementation details:**

    1. **Early return:** If `!spec.imports?.length`, return `{ ok: true, data: spec }`.

    2. **Duplicate `as` check:** Before processing, check for duplicate `as` values in the imports array. If found, return error with type `"merge_error"`.

    3. **For each import in order (first = lowest precedence):**
       - Resolve absolute path: `path.resolve(basedir, imp.ref)` then `path.normalize()`.
       - Check `visited` set for circular detection. If found, return `{ ok: false, errors: [{ type: "circular_import", message with chain, chain array }] }`.
       - Read file with `fs.readFileSync(absPath, "utf-8")`. Catch errors -> return `file_not_found`.
       - Parse with `parse(content)`. If not ok, return `parse_error`.
       - Add normalizedPath to visited set.
       - Recurse: `resolveImportsRecursive(parseResult.data, path.dirname(absPath), visited, [...chain, normalizedPath])`.
       - If recursion fails, return the error.
       - Namespace the resolved spec with `namespaceSpec(resolved.data, imp.as)`.
       - Merge into accumulator: `mergeSpecs(accumulated, namespacedSpec)`.

    4. **Final merge:** Apply local spec (stripped of imports) on top: `mergeSpecs(accumulated, stripImports(spec))`. This ensures local values override all imports.

    5. **Return** `{ ok: true, data: merged }`.

    **Helper: `stripImports(spec)`** -- spread copy without `imports` key. Use destructuring: `const { imports: _, ...rest } = spec; return rest as LogicSpec;`

    **Helper: `namespaceSpec(spec, ns)`** -- namespace record-keyed sections:
    - `steps`: prefix each key with `${ns}.`, also namespace `needs` arrays (each need becomes `${ns}.${need}`), namespace `parallel_steps` arrays, and namespace `branches[].then` values.
    - `decision_trees`: prefix each key with `${ns}.`.
    - Leave all other sections (reasoning, contracts, quality_gates, fallback, global) as-is (they merge by value, not by key).

    **Helper: `mergeSpecs(base, override)`** -- shallow merge per section:
    - Scalar sections (reasoning, contracts, quality_gates, fallback, global): `{ ...base.section, ...override.section }` (override wins).
    - Record sections (steps, decision_trees, nodes): `{ ...base.section, ...override.section }`.
    - Array sections (edges): concatenate.
    - Top-level scalars (spec_version, name, description): override wins.
    - Do NOT merge `imports` (already stripped).
    - Do NOT merge `visual` or `metadata` -- override wins entirely.

    IMPORTANT: The `visited` set and `chain` must use normalized absolute paths consistently. Before the loop starts, add the CURRENT file's normalized path to visited if this is the initial call (or handle via the entrypoint adding the root file).

    Actually, the entrypoint `resolveImports` should NOT add the root to visited -- the root file is the caller, not an import. The visited set tracks imported files. Each import adds itself before recursing.
