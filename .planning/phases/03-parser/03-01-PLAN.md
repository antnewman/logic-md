Create `packages/core/parser.ts` with the following implementation:

**Types (export all four):**
```
ParseError: { message: string; line?: number; column?: number }
ParseSuccess: { ok: true; data: LogicSpec; content: string }
ParseFailure: { ok: false; errors: ParseError[] }
ParseResult: ParseSuccess | ParseFailure
```

**Import pattern:** Use `createRequire(import.meta.url)` to import gray-matter, exactly like schema.ts does for ajv-formats. Cast as `typeof import("gray-matter")`. Import `LogicSpec` from `./types.js` using `import type`.

**parse(input: string): ParseResult implementation:**
1. Guard: if `!input` or `input.trim() === ""`, return ParseFailure with "Input is empty" message
2. Pre-check: if `!matter.test(input)`, return ParseFailure with "No YAML frontmatter found. LOGIC.md files must start with `---`"
3. Try: call `matter(input)`, return ParseSuccess with `data: result.data as LogicSpec` and `content: result.content`
4. Catch: extract line/column from `(error as any).mark` (YAMLException shape -- `mark.line` is 0-based, add 1 for 1-based). Extract reason from `(error as any).reason`. Return ParseFailure with a single ParseError containing the message, line, and column.

**Update index.ts:** Add `export { parse, type ParseResult, type ParseSuccess, type ParseFailure, type ParseError } from "./parser.js";` to the barrel exports.

Do NOT add schema validation here -- that belongs to Phase 4. The parser only extracts and casts.
