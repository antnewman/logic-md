Create `packages/core/dag.ts` implementing the DAG resolver. Follow the research skeleton from 06-RESEARCH.md closely. Key implementation details:

**Types (at top of file):**
```typescript
import type { Step } from "./types.js";

export interface DagError {
  type: "cycle" | "unreachable" | "missing_dependency";
  message: string;
  nodes: string[];
}

export interface DagSuccess {
  ok: true;
  levels: string[][];
  order: string[];
}

export interface DagFailure {
  ok: false;
  errors: DagError[];
}

export type DagResult = DagSuccess | DagFailure;
```

**resolve() function:**
1. Get sorted step names (`Object.keys(steps).sort()`) for deterministic order
2. Handle empty input: return `{ ok: true, levels: [], order: [] }`
3. Validate all `needs` references: check self-references (cycle error), check missing deps (missing_dependency error). Return early if errors found.
4. Build in-degree map and dependents adjacency list
5. Kahn's algorithm with depth tracking:
   - Initialize queue with all steps having in-degree 0, depth 0
   - Sort queue alphabetically before each dequeue for determinism
   - Process: dequeue, add to sorted, update dependents' in-degree and depth
   - Depth of a step = max(depth of all its dependencies) + 1
6. Cycle detection: if `sorted.length < names.length`, remaining nodes are cycle members. Extract cycle path via DFS with gray/black coloring on the cycle-member subgraph.
7. Unreachable detection: forward BFS from roots (steps with no `needs`). Steps in sorted output but not in reachable set (and not cycle members) = unreachable.
8. Group sorted steps by depth into `levels` arrays, sorted alphabetically within each level.
9. Return `{ ok: true, levels, order }` on success.

**extractCyclePath() helper function:**
- Takes cycle member names and steps record
- Uses DFS with gray/black set coloring
- When a gray node is revisited, backtracks to build cycle path
- Returns arrow-separated string (e.g., "A -> B -> C -> A")
- Iterates cycle members in sorted order for determinism

**Critical implementation rules:**
- Do NOT mutate the input `steps` object
- Do NOT throw errors -- return discriminated union results
- Sort alphabetically at every queue insertion point for determinism
- Cycle members must NOT also be reported as unreachable (check cycle set before reporting unreachable)
