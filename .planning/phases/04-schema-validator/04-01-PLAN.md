Create `packages/core/validator.ts` implementing the `validate()` function.

**Function signature**: `validate(fileContent: string): ValidationResult`

The function must be self-contained -- accepts raw file content string (the full .md file with `---` delimiters).

**Implementation steps:**

1. Guard empty/whitespace input: return `{ ok: false, errors: [{ message: "Input is empty", path: "/" }] }`.

2. Check frontmatter delimiters using gray-matter's `.test()`. If missing, return error.

3. Extract frontmatter with gray-matter:
   ```typescript
   const require = createRequire(import.meta.url);
   const matter = require("gray-matter") as typeof import("gray-matter");
   ```
   Use the same CJS interop pattern from parser.ts.

4. Run ajv validation using `createValidator()` from `./schema.js`. Pass `result.data` to the validator.

5. If valid, return `{ ok: true, data: result.data as LogicSpec }`.

6. If invalid, build source position map:
   - Get raw YAML from `result.matter` (gray-matter's raw YAML string without delimiters)
   - Parse with `yaml` package: `parseDocument(yamlSource, { lineCounter })`
   - For each ajv error, resolve the full path:
     - `additionalProperties` keyword: append `params.additionalProperty` to instancePath
     - `required` keyword: append `params.missingProperty` to instancePath
     - Others: use `instancePath` directly
   - Convert path to array: split on `/`, filter empty, map numeric strings to numbers
   - Look up node with `doc.getIn(path, true)` for source position
   - If node not found, try parent path. If still not found, line is undefined.
   - Apply FRONTMATTER_OFFSET of +1 to all line numbers (accounts for opening `---` line)

7. Format error messages using `formatErrorMessage()`:
   - `required`: `Missing required property "X" at Y`
   - `additionalProperties`: `Unknown property "X" at Y`
   - `type`: `Expected {type} at Y`
   - `enum`: `Invalid value at Y. Allowed: a, b, c`
   - Default: `{ajv message} at Y`

8. Return `{ ok: false, errors }`.

**Import pattern** (follow existing codebase conventions):
```typescript
import { createRequire } from "node:module";
import { parseDocument, LineCounter } from "yaml";
import { createValidator } from "./schema.js";
import type { LogicSpec, ValidationError, ValidationResult } from "./types.js";
```
